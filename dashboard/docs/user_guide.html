User Guide

AI Litigation Dashboard의 세부적인 기능과 프리미엄 활용 방법을 안내합니다.

이 앱의 목표: AI 회사들이 허가받지 않은 데이터를 이용하여 AI 모델을 학습시키는 과정에서 발생한 저작권 및 권리 침해 소송들을 지도 기반으로 조회하고 실시간으로 추적하는 것입니다.

1. 🚀 데이터 로드 및 시각화 (Initial Setup)

  • Dataset 선택: 상단 헤더의 CSV Select 드롭다운에서 분석하고자 하는 특정 시점의 데이터셋을 선택합니다.
  • 시각화 갱신: 대시보드의 필터를 수동으로 조정하기 전, Update Visualization 버튼을 눌러 전체 지도와 통계치를 최신 상태로 동기화하십시오.

2. 🗺 인터랙티브 지도 탐색 (Map Navigation)

  • 글로벌 전환: 헤더의 Country 메뉴를 통해 미국(US)과 전 세계(WORLD) 지도를 즉시 전환할 수 있습니다.
  • 초고정밀 줌 (Marquee Zoom):
    • Shift + 드래그: 확대하고 싶은 특정 영역을 사각형으로 드래그하면 정밀 확대됩니다.
    • 마우스 휠: 커서가 위치한 지점을 중심으로 자유로운 확대/축소가 가능합니다.
  • 스마트 툴팁: 지역 위에 마우스를 올리면 해당 지역의 명칭과 실시간 소송 건수가 즉시 표시됩니다.

3. 🎯 고급 다차원 필터링 (Advanced Filtering)

  • 피고(Defendant) 필터: 특정 업체(OpenAI, Meta, Google 등)를 선택하여 집중 분석할 수 있습니다.
  • 학습 데이터(Theme) 필터: 소송의 쟁점이 된 데이터 분야(Music, Art, Code 등)별로 필터링하십시오.
  • 다중 상태 필터: 진행 상태 드롭다운에서 복수의 상태를 자유롭게 선택할 수 있습니다.

4. 📉 통계 및 시기별 추이 분석 (Statistics & Trend)

  • 시기별 소송 트렌드: [STATISTICS] 메뉴 하단의 'Litigation Trend' 차트를 통해 발생 추이를 확인하십시오.
  • 종합 통계: 청구 내용, 피고 점유율, 판결 결과 등 6가지 핵심 차트를 제공합니다.

5. 🔍 고성능 소송 검색 (Power Search)

  • 실시간 통합 검색: 사이드바 상단의 검색창에 키워드를 입력하면 사건명, 법원, 상태를 동시에 검색합니다.
  • 결과 리스팅: 검색 조건에 부합하는 소송들을 세련된 카드 형태로 나열합니다.

6. 🔑 주요 상태값 정의 (Glossary)

  • 준비 중 (Ready): 소송이 제기되었으나 상세 문서가 아직 공개되지 않은 상태.
  • 소 제기됨 (Started): 공식적인 법적 절차와 증거 조사가 개시된 단계.
  • 1심/2심 진행중: 각 심급 법원에서 심판이 진행 중인 상태.
  • 판결 선고 (Decision): 담당 법원의 최종 판결이 내려진 상태.
  • 항소 제기됨 (Appeal): 상급 법원에 항소하여 법적 공방이 진행 중인 소송.
  • 사건 종결 (Closed): 판결 확정, 합의 또는 취하를 통해 마무리된 상태.

7. 🕸️ 리니지 관계 그래프 분석 (Lineage)

복잡한 소송 관계자들 간의 네트워크를 한눈에 파악할 수 있는 고급 시각화 도구입니다.

  • 원고/피고 추적: 주체 간의 연결 고리를 시각적으로 분석합니다.
  • 데이터셋 중심 분석: 특정 데이터셋이 어떤 소송들에 연루되어 있는지 식별합니다.
  • 포커스 모드: 특정 대상을 선택하여 직접적으로 연결된 역사와 관계망만 필터링합니다.
